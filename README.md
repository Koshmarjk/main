# Консольный симулятор экосистемы

Пошаговая симуляция взаимодействия популяций по модели «хищник–жертва».

## Архитектура: три логических слоя

Проект явно разделён на три слоя приложения:

```
┌─────────────────────────────────────────────────────┐
│  СЛОЙ ИНТЕРФЕЙСА (ui.py)                            │
│  Весь вывод в консоль. Получает события от          │
│  бизнес-логики и форматирует их для пользователя.   │
└──────────────────────┬──────────────────────────────┘
                       │ вызывает / получает события
┌──────────────────────▼──────────────────────────────┐
│  БИЗНЕС-ЛОГИКА (ecosystem.py, population.py,        │
│                  organism.py)                        │
│  Логика симуляции. Не знает о консоли.              │
│  Возвращает события строками в интерфейс.           │
└──────────────────────┬──────────────────────────────┘
                       │ читает параметры
┌──────────────────────▼──────────────────────────────┐
│  СЛОЙ ДАННЫХ (data.py)                              │
│  Начальные параметры организмов и конфигурация      │
│  симуляции. Не содержит логики.                     │
└─────────────────────────────────────────────────────┘
```

**Поток вызовов в `main.py`:**
```
main → data.get_*()       # 1. получить данные
     → Ecosystem.simulate_day()  # 2. выполнить логику
     → ui.show_*()        # 3. отобразить результат
```

## Структура файлов

```
life_simulator/
├── main.py        # точка входа, связывает все слои
├── data.py        # слой данных: конфиги и параметры
├── organism.py    # бизнес-логика: Organism (ABC), Herbivore, Predator
├── population.py  # бизнес-логика: Population
├── ecosystem.py   # бизнес-логика: Ecosystem, цикл симуляции
├── ui.py          # слой интерфейса: весь вывод в консоль
└── README.md
```

## Классы

| Класс | Файл | Слой | Описание |
|---|---|---|---|
| `Organism` | organism.py | Бизнес-логика | Абстрактный базовый класс (ABC) |
| `Herbivore` | organism.py | Бизнес-логика | Травоядное, метод `graze()` |
| `Predator` | organism.py | Бизнес-логика | Хищник, методы `hunt()`, `set_target()` |
| `Population` | population.py | Бизнес-логика | Группа особей одного вида |
| `Ecosystem` | ecosystem.py | Бизнес-логика | Оркестратор симуляции, 4 фазы дня |

## Логика одного дня (4 фазы)

1. **Фаза 1** — травоядные пасутся, получают энергию
2. **Фаза 2** — хищники охотятся на травоядных
3. **Фаза 3** — все особи тратят базовую энергию за сутки
4. **Фаза 4** — погибшие (energy = 0) удаляются из популяций

## Запуск

```bash
python main.py
```

Требуется **Python 3.10+**. Внешние зависимости отсутствуют.
